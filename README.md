# DO DAILY.

## アプリケーション名
DO DAILY.  
You already know what to do.

---

## アプリケーション概要
**DO DAILY** は、  
「毎日やるべきことは分かっているのに、続かない」人のための  
**習慣管理 × 振り返り × 目標 × スケジュール** を一体化した自己管理アプリです。

ToDo 管理ではなく、  
**「目標 → 習慣 → 予定 → 振り返り」** を一本の線でつなぎ、  
淡々と積み上げることに集中できる設計を重視しています。

## 利用方法

### ① 習慣（Habit）の登録・実行
1. 新規登録 / ログイン  
2. 習慣を作成（タイトル・内容）  
3. 毎日「Done / Undo」をワンタップで記録  

---

### ② 振り返りの記入・検索
- 朝の振り返り  
- 夜の振り返り  
- 今日の振り返り  
- 週まとめ  

日付・キーワード検索により、  
過去の振り返りを一覧で確認できます。

---

### ③ 道しるべ（5年間の中期目標）
- 年ごとの中期目標を設定
- 表示 / 編集の切り替え
- 日々の行動と将来目標を常に接続

---

### ④ My Schedule（ユーザー予定管理）
Habit に紐づいたスケジュール管理機能です。

#### 特徴
- Habitごとに予定を管理
- 月 / 週 / 日 表示切り替え
- 予定には以下を設定可能  
  - タイトル  
  - 場所  
  - 開始時刻 / 終了時刻  
- 日表示から予定の追加・編集・削除が可能

※ 予定は単なるToDoではなく、  
**Habit（目的）を実現するための行動単位**として設計しています。

---

## アプリケーションを作成した背景
学習・転職活動を進める中で、  
「目標はあるが、日々の行動が曖昧になりやすい」という課題を感じました。

一般的な ToDo 管理ではなく、  
**“行動 → 振り返り → 可視化” を毎日自然に繰り返せる仕組み** が必要だと考え、  
自分自身が使い続けられることを前提に本アプリを開発しました。

## データベース設計（主要モデル）

### User
- has_many :habits  
- has_many :schedule_events  

### Habit
- belongs_to :user  
- has_many :habit_logs  
- has_many :schedule_events  

### ScheduleEvent
- belongs_to :user  
- belongs_to :habit  
- title  
- location  
- starts_at  
- ends_at
